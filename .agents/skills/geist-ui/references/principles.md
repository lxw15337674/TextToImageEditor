# Geist Principles

Geist surfaces should feel precise, quiet, and technical.

## Core characteristics

- Simplicity over ornament.
- High contrast without looking harsh.
- Strong information hierarchy.
- Tight rhythm and disciplined spacing.
- Utility-oriented controls suitable for dashboards, editors, settings, and developer tools.

## Practical interpretation

- Start from typography and spacing, not decoration.
- Make containers legible through subtle layering, borders, and contrast shifts.
- Use one dominant action per area.
- Let dense views stay dense, but keep them readable with alignment and consistent type sizes.

## What to avoid

- Hero sections that look like generic AI landing pages.
- Oversized rounded cards everywhere.
- Washed-out low-contrast text.
- Heavy shadows as the main separation mechanism.
- Accent color sprayed across badges, buttons, tabs, links, and icons simultaneously.
