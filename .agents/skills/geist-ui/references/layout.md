# Layout

Geist layouts feel ordered and deliberate rather than cinematic.

## General structure

- Use consistent page gutters.
- Limit content width intentionally.
- Group related controls tightly.
- Leave more breathing room between sections than between items inside a section.

## Grid behavior

The Geist grid docs emphasize structured layouts, including responsive columns and guide-like alignment. In practice:

- Use grid when alignment helps comprehension, not as a decorative overlay.
- Preserve obvious column logic across breakpoints.
- Let card spans and content groupings communicate importance.
- In marketing sections, use the grid to create tension and asymmetry without losing clarity.

## Page composition

- A title block should establish the page quickly: title, one-sentence support copy, then actions.
- Dense admin and tool views should keep controls close to the data they affect.
- Empty states should be compact and matter-of-fact.
- Tables, settings lists, and editors should prefer horizontal efficiency over oversized padding.

## Responsive guidance

- Collapse complexity, not hierarchy.
- Preserve action visibility on mobile.
- Keep toolbar rows from wrapping into visual chaos; stack deliberately when necessary.
